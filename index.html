Why should I use promises?

A Promise object represents a value that may not be available yet, but will be resolved at some point in future. It allows you to write asynchronous code in a more synchronous fashion.

A common operation, such as reading a .json -file, might look quite different depending on the approach. The more traditional one is to use callbacks, however this often causes an infamous 'pyramamid of doom' or 'callback hell' By using promises, you can:

Turn this...


fs.readFile("file.json", function(err, val) {
    if( err ) {
        console.error("unable to read file");
    }
    else {
        try {
            val = JSON.parse(val);
            console.log(val.success);
        }
        catch( e ) {
            console.error("invalid json in file");
        }
    }
});

... to this


fs.readFileAsync("file.json")
    .then(JSON.parse)
    .then(function(val) {
    console.log(val.success);
})
.catch(SyntaxError, function(e) {
    console.error("invalid json in file");
})
.catch(function(e){
    console.error("unable to read file")
});
